# 메타 광고 택소노미 레퍼런스

> 불변 원본. 클라이언트별 커스터마이징은 `docs/clients/{name}/taxonomy.md`에서.
> 마지막 업데이트: 2026-04-19

---

## 네이밍 체계 개요

```
캠페인명: {MKT목표}_{CPG목표}_{제품코드}_{스프린트}_{YYMMDD}
세트명:   {OPT목표}_{페르소나}_{제품코드}_{타겟}_{지면}_{VP}_{YYMMDD}
광고명:   {USP}_{카피훅}_{소재유형}_{YYMMDD}_{버전}
```

### 설계 원칙

1. **구분자:** 언더스코어(`_`) 고정. 띄어쓰기·특수문자 금지
2. **필드 순서 고정:** 분석 시 split으로 컬럼 자동 분리
3. **고유성 보장:** 날짜 + 스프린트 + 버전 조합으로 중복 방지
4. **그루핑 기준 내장:** 각 필드가 분석 필터 기준이 됨

---

## Level 1 — 캠페인명

```
{MKT목표}_{CPG목표}_{제품코드}_{스프린트}_{YYMMDD}
```

**예시:** `ACQ_AC_MYSVC_S01_260419`

### 필드 정의

| 필드 | 설명 | 코드 목록 |
|------|------|----------|
| MKT목표 | 전체 마케팅 목표 | ACQ(획득) / RET(리텐션) / REV(매출) / BRD(브랜딩) |
| CPG목표 | 캠페인 최적화 목표 | AW(인지) / TR(트래픽) / EN(참여) / LD(리드) / AC(앱설치) / CON(전환) |
| 제품코드 | 광고하는 서비스/제품 | 클라이언트별 정의 (예: MYSVC, APPX, B2BSAS) |
| 스프린트 | 실험 회차 넘버링 | S01, S02, S03... (LMF 탐색 회차) |
| YYMMDD | 캠페인 시작일 | 260419 = 2026년 04월 19일 |

### 분석 그루핑 활용

- **MKT목표별 필터:** ACQ vs RET 성과 비교
- **CPG목표별 필터:** 앱설치(AC) vs 전환(CON) 효율 비교
- **스프린트별 필터:** S01→S02→S03 LMF 발전 추이 추적

## Level 2 — 세트/그룹명

```
{OPT목표}_{페르소나}_{제품코드}_{타겟}_{지면}_{VP}_{YYMMDD}
```

**예시:** `CPI_P1_MYSVC_LA_FEED_VP1_260419`

### 필드 정의

| 필드 | 설명 | 코드 목록 |
|------|------|----------|
| OPT목표 | 광고세트 최적화 목표 | CPI / CPA / CPL / CPM / CTR / ROAS |
| 페르소나 | 타겟 퍼소나 | P1, P2, P3... (taxonomy.md에 정의) |
| 제품코드 | 광고하는 서비스/제품 | 캠페인과 동일 |
| 타겟 | 타겟팅 방식 | LA(유사) / REM(리마케팅) / INT(관심사) / CUS(맞춤) / COLD(콜드) |
| 지면 | 노출 지면 | FEED / REEL / STO(스토리) / ALL / SEARCH |
| VP | LMF 소구점 코드 | VP1~VP5 (taxonomy.md에 정의) |
| YYMMDD | 광고세트 시작일 | 260419 |

### 분석 그루핑 활용

- **페르소나별 필터:** P1 vs P2 타겟 효율 비교
- **타겟별 필터:** LA vs REM 획득 단가 비교
- **지면별 필터:** FEED vs REEL 성과 비교
- **VP별 필터:** 소구점별 반응률 → LMF winner 도출

---

## Level 3 — 광고/소재명

```
{USP}_{카피훅}_{소재유형}_{YYMMDD}_{버전}
```

**예시:** `USP1_PAIN_IMG1x1_260419_v01`

### 필드 정의

| 필드 | 설명 | 코드 목록 |
|------|------|----------|
| USP | 핵심 소구점/USP | USP1~USP5 (VP와 동일하거나 세분화) |
| 카피훅 | 카피 접근 방식 | PAIN(페인포인트) / SOC(사회증거/후기) / QUES(질문형) / STAT(수치/통계) / EMO(감성) / BENE(혜택) |
| 소재유형 | 소재 형식 + 비율 | IMG1x1 / IMG4x5 / IMG9x16 / VID1x1 / VID4x5 / VID9x16 / CAR1x1 |
| YYMMDD | 소재 제작일 | 260419 |
| 버전 | 동일 조건 내 버전 | v01, v02, v03... |

### 분석 그루핑 활용

- **USP별 필터:** USP1 vs USP2 CTR 비교 → winner 소구점 식별
- **카피훅별 필터:** PAIN vs SOC 클릭 패턴 비교
- **소재유형별 필터:** IMG vs VID, 비율별 성과 비교

## 전체 예시 (3레벨 세트)

```
캠페인: ACQ_AC_MYSVC_S01_260419
  └── 세트: CPI_P1_MYSVC_LA_FEED_VP1_260419
        └── 광고: USP1_PAIN_IMG1x1_260419_v01
        └── 광고: USP1_PAIN_IMG4x5_260419_v01
        └── 광고: USP1_SOC_IMG1x1_260419_v01
  └── 세트: CPI_P1_MYSVC_LA_REEL_VP2_260419
        └── 광고: USP2_QUES_VID9x16_260419_v01
```

---

## 고유성 검증 규칙

광고명이 이미 존재하는지 확인:
1. `docs/clients/{name}/taxonomy-log.csv` 조회
2. 동일 문자열 존재 시 → 버전 증가 (v01 → v02)
3. 날짜가 달라도 USP+훅+소재유형이 같으면 버전 누적
