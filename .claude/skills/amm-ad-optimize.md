# /amm-ad-optimize — 광고 최적화

## 트리거

`/amm-ad-optimize` 또는 "광고 최적화", "캠페인 최적화", "성과 개선", "소재 교체"

## 역할

집행 중인 메타 광고 캠페인을 실시간 데이터 기반으로 최적화한다.
예산 배분·소재 교체·타겟 조정 등 액션 플랜을 제시한다.
`/amm-ad-analysis`(종료 후 전체 분석)와 달리 캠페인 **집행 중**에 실행한다.

## Hook 게이트 (→ `.claude/hooks.md` 상세)

**Pre:** GATE-1 (클라이언트 확인)
**Post:** GATE-5 (산출물 저장 확인)

## 독립 실행

선행 스킬 없이 독립 실행 가능.
`output/{name}/setup/` 산출물이 있으면 자동 로드하여 세팅 맥락을 파악한다.

## Sprint Contract (시작 전 합의)

스킬 실행 전 **AskUserQuestion 툴**로 아래를 순서대로 질문한다:

1. "어떤 클라이언트의 캠페인을 최적화할까요? (클라이언트명)"
2. "현재 집행 중인 캠페인명을 알려주세요. (택소노미 기준, 예: ACQ_AC_MYSVC_S01_260419)"
3. "집행 시작일과 현재까지 경과 기간은? (예: 2026-04-19 시작, 7일 경과)"
4. "현재 가장 큰 문제는? (예산 소진 과다 / CTR 저조 / CPA 높음 / 특정 소재 부진 / 기타)"
5. "최근 성과 데이터를 붙여넣어 주세요. (VP별 또는 광고세트별 CTR·CPA·지출 등)"

답변 확인 후 → Phase 0으로 진행.

## 실행 흐름

### Phase 0 — 맥락 로드

```
1. output/{name}/setup/ 최신 폴더 읽기 (있으면)
   → 현재 세팅 구조·소구점 배분 파악

2. docs/clients/{name}/campaign-log.md 읽기 (있으면)
   → 이전 최적화 이력 · 반복 문제 패턴

3. docs/clients/{name}/taxonomy.md 읽기
   → VP 코드 · USP 매핑
```

### Phase 1 — 현황 진단

소구점별 성과 매트릭스:

| VP코드 | 노출 | CTR | CPC | CPA/CPI | 지출 | 판정 |
|--------|------|-----|-----|---------|------|------|
| VP1 | | | | | | |
| VP2 | | | | | | |

판정 기준:
- CTR 업계 평균 이상 + CPA 목표 이하 → **유지**
- CTR 낮음 + CPA 목표 이하 → **타겟 조정 검토**
- CTR 높음 + CPA 목표 초과 → **랜딩 미스매치 의심**
- CTR 낮음 + CPA 목표 초과 → **소구점 교체**

### Phase 2 — 최적화 레버 분석

4가지 레버 중 우선순위 결정:

| 레버 | 현황 | 조치 방향 | 우선순위 |
|------|------|---------|---------|
| 예산 배분 | | winner VP에 집중 | |
| 소재 교체 | | 부진 소재 pause → 신규 투입 | |
| 타겟 조정 | | 오디언스 확대/축소/변경 | |
| 일정 변경 | | 성과 좋은 시간대 집중 | |

### Phase 3 — 액션 플랜

| 우선순위 | 레버 | 현재 상태 | 변경 내용 | 예상 효과 |
|---------|------|---------|---------|---------|
| 1 | | | | |
| 2 | | | | |

> **AI 에이전트 절대 금지:** 메타 광고 관리자 직접 수정·삭제 금지.
> 액션 플랜 출력 후 → 사용자가 직접 집행.

### Phase 4 — 중간 인사이트 기록

`docs/clients/{name}/campaign-log.md`에 append:

```markdown
### [{날짜}] {캠페인명} — 최적화 #{N}회차
- 현황: {요약}
- 문제: {진단}
- 액션: {변경 내용}
- 예상: {기대 효과}
```
