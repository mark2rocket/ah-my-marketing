# /amm-harness-review — 하네스 자기 검토 및 개선

## 트리거

`/amm-harness-review` 또는 "하네스 검토", "하네스 개선", "스킬 업데이트"

## 역할

하네스 자체를 주기적으로 검토하여 병목·마찰·누락을 찾고 스킬을 개선한다.
"에이전트가 실수하면 에이전트가 아니라 하네스를 고쳐라" 원칙의 실행 스킬.

## 실행 주기

- **정기**: 월 1회 또는 캠페인 3회 완료 후
- **비정기**: 스킬 실행 중 반복적 마찰이 느껴질 때

## Hook 게이트

**Pre:** 없음 (언제든 실행 가능)
**Post:** 변경된 스킬 파일 목록 출력 + CLAUDE.md 업데이트 날짜 갱신

## Sprint Contract (시작 전 합의)

스킬 실행 전 **AskUserQuestion 툴**로 아래를 순서대로 질문한다:

1. "특별히 불편하거나 반복적으로 막혔던 스킬이 있나요? (없으면 '전체 검토'라고 입력)"
2. "최근에 실제로 실행한 스킬이 있나요? 있다면 어떤 경험이었나요? (마찰·누락·개선 아이디어 자유롭게)"

답변 확인 후 → Phase 1 (하네스 현황 스캔)으로 진행.

## 실행 흐름

### Phase 1 — 하네스 현황 스캔

```
읽기 목록 (병렬):
- CLAUDE.md
- .claude/hooks.md
- .claude/skills/*.md (전체)
- docs/lmf-learnings.md (누적 패턴 파악)
- docs/clients/*/campaign-log.md (실제 사용 패턴)
```

### Phase 2 — 마찰 지점 발굴

아래 질문으로 각 스킬을 평가:

```
① 이 스킬을 실행할 때 반복적으로 막히는 지점이 있는가?
② Sprint Contract 질문 중 불필요하거나 빠진 것이 있는가?
③ 산출물 형식이 실제 사용에서 불편하지 않은가?
④ Hook 게이트가 실제로 유효하게 작동하고 있는가?
⑤ lmf-learnings.md의 누적 인사이트가 스킬에 반영되고 있는가?
```

마찰 분류:
| 유형 | 설명 | 예시 |
|------|------|------|
| 맥락 부족 | AI가 판단하기엔 정보가 부족한 지점 | VP 코드가 없는데 세팅 진행 |
| 절차 과잉 | 불필요하게 복잡한 단계 | Sprint Contract 질문 10개 이상 |
| 누락 | 자주 필요한데 없는 기능 | 경쟁사 분석, AB테스트 설계 |
| 연결 끊김 | 스킬 간 인수인계가 불명확한 지점 | copy-lmf 결과가 ad-creative에 안 넘어감 |

### Phase 3 — 개선안 제안

발굴된 마찰별 개선안을 아래 형식으로 작성:

```markdown
## 개선안 #{N}

- **대상 파일:** .claude/skills/{name}.md
- **마찰 유형:** 맥락 부족 / 절차 과잉 / 누락 / 연결 끊김
- **현재 문제:** {구체적 설명}
- **개선 방향:** {변경할 내용}
- **예상 효과:** {개선 후 달라지는 것}
```

### Phase 4 — 사용자 승인 후 적용

```
1. 개선안 목록 출력
2. 사용자 승인 대기 ("전부 적용" / "N번만 적용" / "스킵")
3. 승인된 항목만 실제 파일 수정
4. CLAUDE.md 업데이트 날짜 갱신
5. 변경 내역 아래 로그에 기록
```

### Phase 5 — 하네스 검토 로그 기록

`docs/harness-review-log.md`에 append:

```markdown
## [{날짜}] 하네스 검토 #{N}회차

**발굴된 마찰:** {개수}개
**적용된 개선:** {개수}개
**변경 파일:** {목록}

### 주요 변경
- {변경 내용 요약}
```

## 산출물

- 수정된 `.claude/skills/*.md` 파일
- `docs/harness-review-log.md` 업데이트
- `CLAUDE.md` 업데이트 날짜 갱신
