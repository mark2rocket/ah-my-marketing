# /amm-blog — 블로그·뉴스레터 콘텐츠 기획 및 제작

## 트리거

`/amm-blog` 또는 "블로그", "뉴스레터", "롱폼", "아티클"

## 역할

SNS 오가닉 채널용 블로그·뉴스레터 콘텐츠를 기획하고 초안을 작성한다.
구조 설계 → 초안 작성 → Anti-Slop 정제 → 마크다운 산출물.
인스타그램 블로그, 브런치, 서브스택, 자사 블로그 등 롱폼 채널 전용.

## Hook 게이트 (→ `.claude/hooks.md` 상세)

**Pre:** GATE-1 (클라이언트 확인)
**Post:** GATE-5 (산출물 저장 확인)

## 독립 실행

선행 스킬 없이 독립 실행 가능.
`docs/clients/{name}/brand-guide.md`가 있으면 자동 로드.
`lmf-learnings.md`에 누적 인사이트가 있으면 소재로 활용한다.

## Sprint Contract (시작 전 합의)

스킬 실행 전 **AskUserQuestion 툴**로 아래를 순서대로 질문한다:

1. "어떤 클라이언트의 블로그/뉴스레터를 작성할까요? (클라이언트명)"
2. "광고 계정명을 알려주세요. (예: main / kr-performance / default)"
3. "콘텐츠 주제 또는 핵심 메시지를 알려주세요."
4. "채널을 선택해주세요. (인스타 블로그 / 브런치 / 서브스택 / 자사 블로그 / 뉴스레터)"
5. "목표 분량은? (짧게 800자 / 보통 1500자 / 길게 3000자+)"
6. "타겟 독자는 누구인가요? (페르소나 코드 또는 자유 기술)"
7. "참고할 소재가 있나요? (데이터, 사례, 경험담 — 없으면 '없음')"

답변 확인 후 → Phase 0으로 진행.

## 실행 흐름

### Phase 0 — 브랜드·제품 로드 (GATE-0)

```
0. (GATE-0) docs/clients/{name}/brand-guide.md 읽기
   → 없으면: "/amm-brand-setup을 먼저 실행해주세요." → 중단
   → 제품 1개: 자동 선택, 이후 전체 진행에서 이 제품 정보 사용
   → 제품 2~3개: AskUserQuestion으로 선택 요청
     "이번에 광고할 제품/서비스를 선택해주세요: P01 {제품명} / P02 {제품명}"
   → 제품 4개 이상: products/ 폴더 목록 출력 → AskUserQuestion으로 선택 → P{N}.md 로드
```

### Phase 0-1 — 이전 인사이트 로드

```
1. docs/clients/{name}/accounts/{account}/lmf-learnings.md 읽기 (있으면)
   → 고객 언어·winner 소구점 중 이번 주제와 연결할 수 있는 것 파악

2. docs/clients/{name}/brand-guide.md 읽기 (있으면)
   → 브랜드 톤·금지 표현·자주 쓰는 어투 파악
```

없으면: "이전 인사이트 없음. 제공된 정보로 시작합니다."

### Phase 1 — 콘텐츠 구조 설계

채널별 권장 구조:

**인스타 블로그 / 브런치 (스토리텔링형):**
```
1. 훅 — 공감/질문/의외성으로 시작 (1~2문장)
2. 문제 제기 — 독자의 상황 묘사
3. 본론 — 핵심 메시지 (소제목 2~4개)
4. 마무리 — 요약 + 행동 유도
```

**서브스택 / 자사 블로그 (정보형):**
```
1. 요약 — 읽어야 할 이유 한 줄
2. 배경 — 왜 지금 이 주제인가
3. 본론 — 구체적 내용 (소제목 3~5개)
4. 결론 + 다음 액션
```

**뉴스레터:**
```
1. 제목 + 서브제목
2. 이번 호 한 줄 요약
3. 본문 (섹션 2~3개)
4. 오늘의 CTA
```

구조 확정 후 목차 테이블 출력:
| 섹션 | 소제목 | 핵심 내용 | 분량 |
|------|--------|---------|------|
| | | | |

### Phase 2 — 초안 작성 (Generator)

목차 기반으로 전체 초안을 작성한다.

**작성 원칙:**
- 공급자 언어 금지: "~를 제공합니다", "~기능이 있습니다" → 독자 관점 언어로
- 첫 문장이 훅이어야 한다: 독자가 두 번째 문장으로 넘어가게
- 소제목은 질문형 또는 결론형으로: "왜 이게 중요한가?" / "3가지 이유"
- 데이터·사례·인용이 있으면 출처 명시

### Phase 3 — Anti-Slop 정제 패스

> 초안 완성 후 저장 전 반드시 적용.
> 정제 기준: `.claude/rules/creative-design.md` → "카피 정제 필수 패스" (Pass 1~3)

정제 완료 후 → `blog-draft-{YYMMDD}.md` 저장.

### Phase 4 — 발행 체크리스트

- [ ] AI스러운 표현(seamless, 혁신적인, 강력한 등)이 없는가?
- [ ] 첫 문장이 훅으로 작동하는가?
- [ ] 소제목만 읽어도 전체 흐름이 파악되는가?
- [ ] CTA가 명확하게 있는가?
- [ ] 목표 분량을 충족했는가?
- [ ] 브랜드 톤과 일치하는가?
