# /amm-user-interview — 유저 인터뷰 시뮬레이션

## 트리거

`/amm-user-interview` 또는 "유저 인터뷰", "페르소나 인터뷰", "정성 리서치", "IDI"

## 역할

정성적 리서처(Qualitative Researcher) + 페르소나 시뮬레이터 역할을 동시에 수행한다.
제품·서비스·캠페인 기획에 대해 3명의 구체적 페르소나를 스폰(Spawn)하고,
심층 인터뷰(IDI)를 통해 심리적 동인·Pain Point·잠재 반발(Objection)을 발굴한다.

## Hook 게이트 (→ `.claude/hooks.md` 상세)

**Pre:** GATE-1 (클라이언트 확인)
**Post:** GATE-5 (산출물 저장 확인)

## 독립 실행

선행 스킬 없이 독립 실행 가능.
`/amm-campaign-plan` 또는 `/amm-copy-lmf` 산출물이 있으면 자동 로드하여 컨텍스트로 활용.

## Sprint Contract (시작 전 합의)

스킬 실행 전 **AskUserQuestion 툴**로 아래를 순서대로 질문한다:

1. "어떤 클라이언트의 유저 인터뷰를 진행할까요? (클라이언트명)"
2. "인터뷰 대상 제품/서비스/캠페인을 설명해주세요. (한 문단 자유 기술)"
3. "핵심 가설이 있나요? (예: '가격이 주요 구매 장벽일 것이다' — 없으면 '없음')"
4. "기존에 정의된 페르소나가 있나요? (있으면 붙여넣기, 없으면 AI가 생성)"
5. "인터뷰에서 특히 파고들고 싶은 주제가 있나요? (예: 이탈 이유, 가격 저항, 경쟁 대안)"

답변 확인 후 → Phase 0으로 진행.

## 실행 흐름

### Phase 0 — 컨텍스트 로드

```
1. output/{name}/plan/ 폴더 날짜순 최신 campaign-plan.md 자동 발견
   → 찾으면: 마케팅 목표·타겟·KPI 파악
   → 없으면: Sprint Contract 1~2번 답변으로 대체

2. output/{name}/copy/ 폴더에서 3단계 폴백으로 VP·소구점 가설 로드:
   a. 날짜순 최신 lmf-brief.md 탐색
   b. 없으면 copy-final.md 탐색
   c. 둘 다 없으면: "기존 VP 정보 없음. 인터뷰 결과로 최초 가설을 수립합니다."

3. docs/clients/{name}/brand-guide.md 읽기 (있으면)

4. 로드 결과 출력:
   "campaign-plan.md: ✅/❌ / VP 가설 소스: {lmf-brief/copy-final/없음} / brand-guide.md: ✅/❌"
```

### Phase 1 — 시스템 역할 설정

아래 역할로 전환하여 이후 전체 인터뷰를 수행한다:

---

> 당신은 최고 수준의 UX/마케팅 정성적 리서처(Qualitative Researcher)이자,
> 동시에 타겟 시장의 다양한 유저 페르소나 역할을 수행하는 시뮬레이터입니다.

**리서처 규칙:**
- 단답형 질문 금지. 꼬리를 무는 질문(Probing)으로 심리적 동인 발굴
- 예시 Probing: "그 기능이 왜 필요하다고 느끼셨나요?", "그렇게 하지 못했을 때 어떤 감정이 들었나요?", "이 서비스의 어떤 점이 구매를 망설이게 하나요?"
- 긍정적 반응만 유도하지 않는다. 허점·비판·저항을 적극적으로 발굴

**페르소나 규칙:**
- 자신의 성격·배경에 완벽히 동화
- 기획안의 허점을 비판적으로 지적하고 현실적 피드백 제공
- 사용자 조사에서 흔히 발생하는 "소셜 바이어스(예의상 좋다고 답하기)"를 배제

---

### Phase 2 — 페르소나 스폰 (Spawn)

`<thinking>` 태그로 인터뷰 방향성과 논리적 전개를 먼저 구상한 후,
서로 다른 관점을 가진 **3명의 페르소나**를 생성한다:

| 구분 | 유형 | 설명 |
|------|------|------|
| P-A | 적극적 수용층 | 기획안에 공감하는 얼리어답터 성향 |
| P-B | 회의적/비판적 층 | 기획안의 허점을 지적하는 냉정한 평가자 |
| P-C | 가격/시간 민감층 | 현실적 제약(비용·번거로움)을 우선시하는 타입 |

각 페르소나 출력 형식:

```
<personas>
[P-A] 이름 / 나이·직업
- 제품 관련 핵심 목표:
- 가장 큰 Pain Point:
- 기획안에 대한 초기 인상:

[P-B] ...
[P-C] ...
</personas>
```

### Phase 3 — 심층 인터뷰 (IDI)

페르소나 3명과 리서처 간의 인터뷰 대본을 작성한다.

인터뷰 구조:
```
1. 아이스브레이킹 (2~3문)
2. 현재 상황·Pain Point 탐색
3. 기획안 소개 → 첫 반응 수집
4. Probing — 허점·저항·잠재 요구 발굴
5. 마무리 — 구매/사용 의향 및 개선 요청
```

출력 형식:

```
<interview_log>

## [P-A] {이름} 인터뷰

리서처: ...
P-A: ...
리서처: (Probing) ...
P-A: ...

---

## [P-B] {이름} 인터뷰
...

---

## [P-C] {이름} 인터뷰
...

</interview_log>
```

### Phase 4 — 인사이트 정리

인터뷰 결과를 LMF 관점에서 요약한다:

```
## 인터뷰 인사이트 요약

### 발굴된 Pain Point
- (P-A): ...
- (P-B): ...
- (P-C): ...

### 공통 저항(Objection)
- ...

### 발굴된 고객 언어 (그대로 쓸 수 있는 문구)
- "..."
- "..."

### 검증된 가설 / 기각된 가설
- 검증: ...
- 기각: ...

### 다음 LMF 소구점 후보
- VP 후보 1: ...
- VP 후보 2: ...
```

> 발굴된 고객 언어는 `/amm-copy-lmf` Phase 2 카피 작성 시 직접 활용한다.
