# /amm-copy-lmf — LMF 기반 광고 카피 기획

## 트리거

`/amm-copy-lmf` 또는 "카피 기획", "LMF", "광고 문구", "소구점 발굴"

## 역할

LMF(Language Market Fit) 4단계 프로세스를 통해 고객의 언어를 찾아
광고 카피·소구점·메시지를 발굴한다. Generator/Evaluator 분리 구조.

## Hook 게이트 (→ `.claude/hooks.md` 상세)

**Pre:** GATE-1 (클라이언트 확인)
**Phase 3 진입 전:** GATE-4 (Generator/Evaluator 분리 확인)
**Post:** GATE-5 (산출물 저장) → GATE-6 (lmf-learnings.md 업데이트)

## LMF 핵심 원칙

> "이제는 우리가 하고자 하는 말 하지 마세요. 고객이 원하는 말을 해야 한다."

- 공급자 언어(설명) → ❌ "~기능이 있습니다", "~을 제공합니다"
- 고객 언어(설득) → ✅ 고객의 Pain Point·욕구·상황 언어

**메시지 도출 체인:** 퍼소나의 니즈 → 브랜드 VP → 소구점 가설 → 광고 카피 → 소재

## Sprint Contract (시작 전 합의)

스킬 실행 전 **AskUserQuestion 툴**로 아래를 순서대로 질문한다:

1. "어떤 클라이언트의 LMF 카피를 기획할까요? (클라이언트명)"
2. "제품/서비스를 한 줄로 설명해주세요."
3. "현재 LMF 어느 단계에서 시작할까요? (1단계: 퍼소나 정의 / 2단계: 소구점 발산 / 3단계: 소재 고도화 / 4단계: 전채널 적용)"
4. "기존 카피 데이터가 있나요? (있으면 성과 데이터와 함께 붙여넣어 주세요)"
5. "이번에 발굴할 목표 소구점 개수는 몇 개인가요? (권장: 3~5개)"

답변 확인 후 → Phase 0 (이전 인사이트 로드)으로 진행.

## 실행 흐름 (4단계)

---

### Phase 0 — 이전 인사이트 로드 (Compounding)

**매 실행 시 가장 먼저 수행.**

```
1. docs/lmf-learnings.md 읽기
   → 이 클라이언트의 이전 winner 소구점이 있는가?
   → 반복적으로 실패한 소구점 패턴이 있는가?
   → 발굴된 고객 언어 중 재활용할 수 있는 것은?

2. docs/clients/{name}/campaign-log.md 읽기
   → 파일 존재하는 경우: 활성 소구점 현황 + 기각된 VP 코드 확인
   → 파일 없는 경우 (세션 횟수별 분기):
      - lmf-learnings.md에 이 클라이언트 데이터가 있으면:
        "campaign-log.md가 없습니다. lmf-learnings.md 데이터만으로 진행합니다."
      - 둘 다 없으면:
        "첫 번째 LMF 세션입니다. 제로에서 시작합니다."

3. output/{name}/plan/ 폴더에서 날짜순 최신 campaign-plan.md 자동 발견
   → 찾으면: MKT목표·KPI·타겟 페르소나 로드 → Step 1-1 퍼소나 정의에 활용
   → 없으면: Sprint Contract에서 사용자에게 직접 질문 (기존 방식 유지)

4. 요약 출력 (3가지 케이스):
   [케이스 A — 풍부한 데이터]
   "총 {N}회 캠페인 데이터가 있습니다.
    이전 winner: VP{N}({소구점}). 고객 언어: '{문구}'
    기각된 소구점: VP{N}({이유})
    이번 세션은 여기서 이어갑니다."

   [케이스 B — lmf-learnings만 있음]
   "campaign-log.md 없음. lmf-learnings.md 기반으로 진행합니다.
    확인된 이전 인사이트: {내용}
    campaign-log.md는 /amm-ad-analysis 완료 후 자동 생성됩니다."

   [케이스 C — 첫 번째 세션]
   "첫 번째 LMF 세션입니다. 제로에서 시작합니다."
```

### 1단계: LMF 준비하기

**목표:** 퍼소나 + VP + 소구점 가설 수립

#### Step 1-1. 퍼소나 정의

```
다음을 정의한다:
- 주요 타겟 (누구인가?)
- 핵심 Pain Point (어떤 문제를 갖고 있는가?)
- 현재 대안 (지금은 어떻게 해결하고 있는가?)
- 우리 제품을 사용하는 이유 (Why?)
- 사용 빈도 (Frequency)
```

#### Step 1-2. Use Case Map 작성

```
Use Case | Persona | Problem | Alternative | Why | Frequency
```

#### Step 1-3. VP(Value Promise) 도출

3가지 축으로 정의:
- **차별화**: 왜 우리 서비스여야 하는가?
- **문제해결**: 고객의 문제를 어떻게 해결하는가?
- **니즈만족**: 고객의 욕구를 어떻게 충족하는가?

#### Step 1-4. 소구점 가설 수립

가설 작성 공식:
```
"[퍼소나의 어떤 문제]에 맞춘 [특정 VP]를 소구하기 위해,
[어떤 고객의 언어]로 소구하면 더 많은 고객을 획득할 것이다"
```

유의사항:
- 가설은 MECE (중복·누락 없이)
- 결과를 지표로 나타낼 수 있어야 함
- 단독으로 실험 가능해야 함

---

### 2단계: LMF 본격 찾기 (발산형)

**목표:** 많은 소구점 가설을 캠페인으로 검증, winner 소구점 발굴

**GENERATOR 모드** — 이 단계에서는 평가 없이 생성만

```
각 소구점 가설별로:
1. 메시지 아이데이션 (퍼소나 × VP × 소구점 × 문구)
2. 헤드라인 5개 이상
3. 본문 카피 3개 이상
4. CTA 3가지 이상
```

메시지 아이데이션 테이블:
| 퍼소나 | VP | 메인 소구점 | 문구 |
|--------|-----|------------|------|
| | | | |

#### 필수 — Anti-Slop 정제 패스

> 카피 초안 생성 후 저장 전 반드시 적용.
> 정제 기준: `.claude/rules/creative-design.md` → "카피 정제 필수 패스" (Pass 1~3)

정제 완료 후 → `copy-candidates-{YYMM}.md` 저장 → GATE-4 통과

---

### 3단계: LMF 소재 고도화 (수렴형)

**목표:** 발산형에서 찾은 winner 소구점을 다양한 크리에이티브로 고도화

**EVALUATOR 모드** — 별도 패스에서 ICE 채점

#### ICE 우선순위 채점

| 소구점 | Impact (TAM 영향력, /10) | Ease (제작 난이도, /10) | ICE 점수 |
|--------|--------------------------|------------------------|----------|
| VP1 | | | |
| VP2 | | | |

**Impact 기준:**
- 10점: TAM 100% 예상, 소구점이 100% 직관적으로 이해됨
- 5점: TAM 50% 예상, 소구점이 50%만 이해되어도 성공
- 1점: TAM 10% 예상, 소구점이 10%만 이해되어도 성공

**Ease 기준:**
- 10점: 당일 바로 제작 가능 (기존 크리에이티브 활용)
- 5점: 2주 투자 필요 (새 크리에이티브/광고 포맷)
- 1점: 2개월 투자 필요 (촬영/폼이 많이 들어가는 콘텐츠)

#### 소재 베리에이션 설계

검증된 소구점에 대해:

| 베리에이션 타입 | 변수 | 예시 |
|----------------|------|------|
| 포맷 | 이미지/영상/카루셀 | IMG1x1 vs VID9x16 |
| 타겟팅 | 연령/성별/맞춤 | F2534 vs M2534 |
| 에셋 | BG컬러/CTA문구/카피위치 | CTA A vs CTA B |
| 워딩 | 고객친화어/도치법/동음이의어 | 직접화법 vs 리뷰체 |

---

### 4단계: 발굴한 LMF 적용

**목표:** 찾아낸 고객 언어를 전 채널·전 접점에 적용

적용 범위:
- 광고 소재 (카피, 크리에이티브)
- 랜딩 페이지
- 앱/서비스 온보딩
- CRM 메시지
- 팀 내부 문화 전파 가이드

## Generator/Evaluator 분리 규칙

> **절대 규칙:** 같은 세션에서 생성과 평가를 동시에 하지 않는다.

```
Session A (Generator):
→ 소구점 가설 생성
→ 카피 초안 작성
→ [파일 저장 후 세션 종료]

Session B (Evaluator):
→ 저장된 카피를 새 컨텍스트에서 읽기
→ ICE 채점
→ 우선순위 정렬
→ 최종 선별
```
