# /amm-landing-audit — 랜딩페이지 LMF 감사

## 트리거

`/amm-landing-audit` 또는 "랜딩 감사", "랜딩페이지 분석", "LP 점검", "랜딩 카피 리뷰"

## 역할

광고와 랜딩페이지 간 메시지 일관성을 LMF 관점에서 감사한다.
"CTR은 높은데 전환이 안 된다"는 문제의 주요 원인인 광고↔랜딩 메시지 미스매치를 진단하고 개선안을 제시한다.

## Hook 게이트 (→ `.claude/hooks.md` 상세)

**Pre:** GATE-1 (클라이언트 확인)
**Post:** GATE-5 (산출물 저장 확인)

## 독립 실행

선행 스킬 없이 독립 실행 가능. `/amm-ad-analysis` 결과와 함께 실행하면 가장 효과적.

## Sprint Contract (시작 전 합의)

스킬 실행 전 **AskUserQuestion 툴**로 아래를 순서대로 질문한다:

1. "어떤 클라이언트의 랜딩페이지를 감사할까요? (클라이언트명)"
2. "광고 계정명을 알려주세요. (예: main / kr-performance / default)"
3. "랜딩페이지 URL 또는 텍스트를 제공해주세요."
4. "연결된 광고의 카피(헤드라인·본문)를 알려주세요. (여러 개면 모두 제공)"
5. "현재 성과 문제가 있나요? (예: CTR 높은데 전환율 낮음 / 이탈률 높음 / 특정 VP만 전환 안 됨)"
6. "Primary Metric은 무엇인가요? (CPI / CPA / 회원가입 / 문의 등)"

답변 확인 후 → Phase 0으로 진행.

## 실행 흐름

### Phase 0 — 선행 산출물 자동 로드

```
0. (GATE-0) docs/clients/{name}/brand-guide.md 읽기
   → 없으면: "/amm-brand-setup을 먼저 실행해주세요." → 중단
   → 제품 1개: 자동 선택, 이후 전체 진행에서 이 제품 정보 사용
   → 제품 2~3개: AskUserQuestion으로 선택 요청
     "이번에 광고할 제품/서비스를 선택해주세요: P01 {제품명} / P02 {제품명}"
   → 제품 4개 이상: products/ 폴더 목록 출력 → AskUserQuestion으로 선택 → P{N}.md 로드

1. docs/clients/{name}/accounts/{account}/taxonomy.md 읽기 (있으면)
   → VP·USP 코드 로드 → 메시지 매핑 테이블에서 코드 기준으로 분류

2. output/{name}/{account}/analysis/ 폴더 날짜순 최신 analysis.md 자동 발견
   → 찾으면: VP별 CTR·CPA 성과 로드 → "성과 데이터 기반 감사 모드" 진행
     ("CTR은 높은데 CPA 높음 → 랜딩 미스매치" 패턴 우선 진단)
   → 없으면: "성과 데이터 없음. 메시지 일관성만으로 감사합니다."

3. 로드 결과 출력:
   "taxonomy.md: ✅/❌ / analysis.md: ✅(성과 기반 모드)/❌(메시지 전용 모드)"
```

### Phase 1 — 광고↔랜딩 메시지 매핑

광고에서 사용한 소구점과 랜딩페이지 카피를 나란히 배치:

| 구분 | 광고 메시지 | 랜딩 메시지 | 일치 여부 |
|------|-----------|-----------|---------|
| 헤드라인 | | | ✅ / ❌ |
| 핵심 소구점 (VP) | | | ✅ / ❌ |
| CTA | | | ✅ / ❌ |
| 타겟 페르소나 언어 | | | ✅ / ❌ |

### Phase 2 — LMF 관점 진단

#### 2-1. 공급자 언어 vs 고객 언어 점검

랜딩페이지 카피를 문장 단위로 분류:

```
공급자 언어 (개선 필요):
- "~를 제공합니다" → 개선: "..."
- "~기능을 갖추고 있습니다" → 개선: "..."

고객 언어 (유지):
- "..."
```

#### 2-2. 미스매치 패턴 진단

흔한 미스매치 유형 체크:

| 패턴 | 설명 | 해당 여부 |
|------|------|---------|
| 소구점 불일치 | 광고는 PAIN, 랜딩은 BENE 위주 | ○/× |
| 톤 불일치 | 광고는 감성적, 랜딩은 딱딱한 설명체 | ○/× |
| 페르소나 불일치 | 광고 타겟과 랜딩 가정 타겟이 다름 | ○/× |
| CTA 불일치 | 광고 CTA와 랜딩 CTA 방향이 다름 | ○/× |
| 정보 과부하 | 랜딩에서 너무 많은 것을 설명 | ○/× |
